# カラーロール（色の役割）ガイド

> Serendie Design System の [Color Role](https://serendie.design/foundations/color/role/) を参考に、当プロジェクトのセマンティックカラートークンのロール体系をまとめたもの。

## 概要

カラーロールとは、カラーパレット（プリミティブトークン）から選んだ色に **デジタルプロダクト内での意味や役割** を持たせたもの。Material Design 3 の設計をベースに簡素化したサブセットとなっている。

ロールは **3つの区分** に分かれる:

| 区分 | 目的 | 例 |
|------|------|-----|
| **Impression**（印象） | ブランドやフィードバックの印象を伝える | Primary, Positive, Negative |
| **Component**（構成） | UI の基盤構造を形作る | Surface, Outline |
| **Interaction**（インタラクション） | 操作状態の変化を表現する | Hovered, Selected, Disabled |

## 前景・背景の分離パターン（on / Container）

アクセシブルなUI設計とダークモード対応のため、**前景色（テキスト・アイコン）と背景色を分離して定義する**。

### ルール

| 接頭辞/接尾辞 | 意味 | 例 |
|--------------|------|-----|
| （なし） | ロール色そのもの | `primary.DEFAULT` |
| `on` | そのロール色の **上に載せる** 前景色 | `primary.on` |
| `container` | 大面積の背景色（ロール色より控えめ） | `primary.container` |
| `onContainer` | container の **上に載せる** 前景色 | `primary.onContainer` |

### なぜ分離するのか

1. **アクセシビリティ**: `DEFAULT` と `on` のペア、`container` と `onContainer` のペアでコントラスト比を保証できる
2. **ダークモード**: ライト/ダークで背景と前景の値を独立に切り替えられる
3. **設計意図の明示**: 「この色は背景に使う」「この色はテキストに使う」が名前から自明になる

### DEFAULT と container の使い分け

```
┌─────────────────────────────────────────────────┐
│  DEFAULT / on                                   │
│  → 小面積・高強調の要素に使う                      │
│    例: CTAボタン塗り、FAB、選択中タブのインジケーター   │
│                                                 │
│  container / onContainer                        │
│  → 大面積・低強調の背景に使う                      │
│    例: Badge背景、Chip背景、Alert背景、カード背景     │
└─────────────────────────────────────────────────┘
```

- `primary.DEFAULT` (= 鮮やかなブランド色) をボタン全面に塗ると強い印象になる
- `primary.container` (= 控えめなブランド色) をバッジや通知の背景に使うと落ち着いた印象になる

## Impression（印象）

ユーザーが受ける「印象」に関連するロール。ブランドカラーとUIステータスカラーの2系統がある。

### ブランドカラー

| ロール | 用途 | 具体的なUI例 |
|--------|------|-------------|
| `primary` | プロダクトのブランドを象徴し、UIの重要度を示す | CTAボタン、FAB、アクティブなタブ・トグル、リンクテキスト、プログレスバー |
| `secondary` | primary を補助する | セカンダリボタン、選択中のフィルターチップ、展開中のFABメニュー |
| `tertiary` | primary/secondary に続く第三の色 | 三段階の優先度表現が必要な場面のみ（通常は使用頻度が低い） |

### UIステータスカラー

| ロール | 用途 | 具体的なUI例 |
|--------|------|-------------|
| `positive` | 成功・完了・安全 | 成功バナー、完了チェック、有効ステータスバッジ |
| `negative` | エラー・失敗・危険 | エラーメッセージ、バリデーション警告、削除ボタン、必須マーク |
| `notice` | 警告・注意喚起 | 警告バナー、残り少ない在庫表示、非推奨機能の告知 |

### バリエーション一覧

全 Impression ロール共通で以下の4バリエーションを持つ。UIステータスカラー（positive / negative / notice）はさらに `containerVariant` / `onContainerVariant` を持つ。

| バリエーション | 用途 | 使用例 |
|--------------|------|-------|
| `DEFAULT` | ロール色そのもの。小面積・高強調の要素 | ボタン塗り、アイコン色、テキストリンク |
| `on` | DEFAULT の上に載せる前景色 | ボタン内ラベル、ボタン内アイコン |
| `container` | 大面積の背景色。DEFAULT より控えめ | Alert の背景、Badge の背景、カード背景 |
| `onContainer` | container の上に載せる前景色 | Alert 内テキスト、Badge 内ラベル |
| `containerVariant` | container の別バリエーション（より控えめ） | 薄い背景のインライン通知 |
| `onContainerVariant` | containerVariant の上に載せる前景色 | インライン通知内テキスト |

## Component（構成）

UI 要素の形成に用いるロール。ニュートラルカラーベースで、ページ全体の骨格を構成する。

### Surface（面）

ページやカードなどの **背景面** を構成するロール。

| バリエーション | 用途 | 具体的なUI例 |
|--------------|------|-------------|
| `DEFAULT` | ページのデフォルト背景色 | `<body>` 背景、メインコンテンツ背景 |
| `on` | surface 上のメインテキスト色 | 見出し、本文テキスト |
| `onVariant` | surface 上のサブテキスト色（on より控えめ） | キャプション、プレースホルダ、補助テキスト |
| `dim` | 暗めの surface | セクション区切りの背景、ストライプテーブルの偶数行 |
| `bright` | 明るめの surface | フローティング要素の背景、ポップオーバー |
| `container` | コンテナ要素の背景 | カード背景、サイドバー背景、入力フィールド背景 |
| `containerBright` | 明るめのコンテナ背景 | ネストされたカード内の区画 |
| `containerDim` | 暗めのコンテナ背景 | アクティブなリスト項目、選択されたエリア |
| `inverse` | 反転背景（ライトモードなら暗い色） | Snackbar 背景、Tooltip 背景 |
| `inverseOn` | inverse 上のテキスト色 | Snackbar テキスト、Tooltip テキスト |
| `inversePrimary` | inverse 上のブランドカラー | Snackbar 内のアクションリンク |

### Outline（線）

ボーダーや区切り線に使うロール。

| バリエーション | 用途 | 具体的なUI例 |
|--------------|------|-------------|
| `DEFAULT` | 標準のボーダー色 | 入力フィールドのボーダー、カードのボーダー |
| `bright` | 控えめなボーダー色（DEFAULT より薄い） | Divider、セクション区切り線 |
| `dim` | 強調されたボーダー色（DEFAULT より濃い） | フォーカスリング、アクティブなボーダー |

### Scrim（幕）

| トークン | 用途 | 具体的なUI例 |
|---------|------|-------------|
| `scrim` | 半透明オーバーレイ | モーダルの背景幕、ドロワーの背景幕 |

## Interaction（インタラクション）

UIコンポーネントの **状態変化** を表現するロール。半透明色を **状態レイヤー** として要素の上に重ねる方式で実現する。

> 状態レイヤー方式: 背景色を直接変えるのではなく、半透明色を上から重ねることで、元の色のニュアンスを保ちながら状態を表現する。

### Hovered（ホバー）

| バリエーション | 用途 | 具体的なUI例 |
|--------------|------|-------------|
| `DEFAULT` | 標準のホバーレイヤー（やや濃い半透明） | リスト項目のホバー、テーブル行のホバー |
| `variant` | 控えめなホバーレイヤー（薄い半透明） | ナビゲーション項目のホバー |
| `onPrimary` | primary 色の上でのホバーレイヤー | ブランドカラーボタンのホバー |

### Selected（選択）

| バリエーション | 用途 | 具体的なUI例 |
|--------------|------|-------------|
| `DEFAULT` | 選択状態のレイヤー | チェックボックスの選択行、データテーブルの選択行 |
| `surface` | 選択状態の背景色（不透明） | タブの選択状態背景、サイドバーのアクティブ項目 |

### Disabled（非活性）

| バリエーション | 用途 | 具体的なUI例 |
|--------------|------|-------------|
| `DEFAULT` | 非活性要素の背景色 | 無効化されたボタン背景、無効化された入力フィールド |
| `onSurface` | 非活性要素上のテキスト色 | 無効化されたボタンのラベル、無効化されたリンク |

## ロール選択のフローチャート

```
この色は何の目的で使う？
│
├─ ブランドの印象を伝えたい → primary / secondary / tertiary
│
├─ 成功/失敗/警告を伝えたい → positive / negative / notice
│
├─ UIの基盤（背景・テキスト・線）
│   ├─ 面（背景）→ surface.*
│   ├─ テキスト → surface.on / surface.onVariant
│   ├─ 線（ボーダー）→ outline.*
│   └─ オーバーレイ幕 → scrim
│
└─ 操作状態を表現したい
    ├─ ホバー → hovered.*
    ├─ 選択 → selected.*
    └─ 非活性 → disabled.*
```
