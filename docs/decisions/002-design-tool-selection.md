# ADR-002: デザインツールの選定

- ステータス: 承認済み
- 日付: 2026-02-22

## コンテキスト

デザインシステムのトークン定義とコンポーネント設計を行うデザインツールを選定する必要がある。以下の前提条件がある:

- チームは開発者のみで構成されている
- トークンは Panda CSS + TypeScript でコードファーストに管理している（ADR-001）
- コンポーネントは Ark UI ベースで React 実装する
- Git ベースのワークフローを優先する

事前に Pencil / Figma のデザインツール比較および移行パスの調査を行った（[調査結果](../research/pencil-vs-figma.md)）。

## 決定

**Pencil（IDE ネイティブ設計ツール）** をデザインツールとして採用する。

### 選定理由

| 判断軸 | Pencil | Figma |
|--------|--------|-------|
| コードファーストとの親和性 | 高い。TypeScript トークンを直接インポート可能 | 低い。Tokens Studio 等の中間ツールが必要 |
| Git 統合 | ネイティブ。`.pen` ファイルを PR でレビュー可能 | 外部ツール経由。デザイン変更はコードレビューに乗らない |
| 開発者の学習コスト | 低い。IDE 内で完結 | 中程度。別環境の操作を習得する必要がある |
| AI 支援 | コア機能。MCP で Claude Code が直接操作 | 補助的。コード生成は別の関心事 |
| コスト | 無料（早期アクセス中） | $36〜75/月（3〜5人） |

### ワークフロー

```
TypeScript トークン定義 (src/tokens/)    ← source of truth
    ↓
Panda CSS config (panda.config.ts)
    ↓
styled-system/
    ↓
Pencil (.pen) でコンポーネント設計 → AI がコード生成
    ↓
Git commit → PR レビュー（デザイン + コード）
```

### .pen ファイルの管理方針

- `.pen` ファイルはリポジトリにコミットする
- コンポーネント設計の `.pen` ファイルは `src/components/` 配下に配置する
- 命名・レイヤー階層を統一し、将来の Figma 移行時にプラグインが正しく読み取れるようにする

### Figma への移行パス

Pencil のロックインリスクを低減するため、以下を維持する:

1. **トークンはコードが source of truth**: Pencil にも Figma にも依存しない
2. **W3C DTCG 対応を意識**: Style Dictionary による変換が可能な構造を保つ
3. **移行判断のタイミング**: コンポーネント数が 5〜10 程度の段階で再評価する

移行が必要になった場合:
- トークン: W3C DTCG JSON 経由で Figma Variables にインポート可能（自動化可能）
- ビジュアル: コミュニティ製 Figma プラグインで `.pen` → Figma 変換可能
- コンポーネント: React 実装を仕様として Figma で再構築（手動作業が中心）

## 根拠

| 判断 | 選択 | 理由 |
|------|------|------|
| ツール | Pencil | 開発者のみのチームで、コードファーストのワークフローに最も適合する |
| Figma 不採用 | 現時点では不要 | 非開発者との協業がなく、Figma のビジュアル協業機能の恩恵が小さい |
| 移行戦略 | 出口戦略を事前に策定 | Pencil は新しいツールのため、Figma への移行パスを確保しておく |
| .pen の管理 | Git にコミット | デザインとコードを同一の PR でレビュー可能にする |
