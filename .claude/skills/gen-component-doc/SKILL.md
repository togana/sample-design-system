argument-hint: <コンポーネント名>
---

$ARGUMENTS で指定されたコンポーネントのソースコードを読み、ドキュメントを生成してください。

## 対象ファイルの読み込み

`src/components/{name}/` 配下の以下のファイルをすべて読む:

- `{name}.tsx` — メインコンポーネント（Props、状態管理、アクセシビリティ）
- `{name}.recipe.ts` — Panda CSS レシピ（バリアント、サイズ、デフォルト値）
- `index.ts` — エクスポート内容の確認

## 出力先

`docs/components/{name}.md`

既存ファイルがある場合は上書きして更新する。

## 出力フォーマット

以下のフォーマットに厳密に従う:

```markdown
# {ComponentName}

## 概要
（1-2文で説明）

## いつ使うか
- ...

## いつ使わないか
- ...

## Props
| Prop | 型 | デフォルト | 説明 |
|------|-----|-----------|------|
| ...  | ... | ...       | ...  |

## バリアント
### variant
（各バリアントの用途を説明）

### size
（各サイズの用途を説明）

## コード例

### 基本
（コード）

### バリアント一覧
（コード）

### 無効化
（コード）

## Do / Don't
| ✅ Do | ❌ Don't |
|-------|---------|
| ...   | ...     |

## アクセシビリティ
- キーボード操作
- ARIA属性
- フォーカス管理
```

## 記述ルール

### Props テーブル

- ソースコードの型定義から正確に抽出する
- デフォルト値は `defaultVariants` や destructuring のデフォルト値から取得する
- `<button>` 等の標準 HTML 属性は個別に列挙せず、末尾に「標準属性をすべてサポート」と記載する

### バリアント

- レシピの `variants` と `compoundVariants` から各バリアントの視覚的特徴を説明する
- 具体的な使用場面を添える

### コード例

- import パスは `@/components/{name}` 形式にする
- 実際にコンパイルが通るコードを書く
- コンポーネントが持つ主要な機能（アイコン、ローディング等）ごとにセクションを追加する

### Do / Don't

- ソースコードの実装（アクセシビリティ対応、状態管理など）から導出する
- 最低4行は記載する

### アクセシビリティ

- ソースコードで使用している ARIA 属性を正確に記載する
- キーボード操作はネイティブ要素の標準動作も含める
- フォーカスリングのスタイル仕様を記載する

## 作業手順

1. ソースコードを読む
2. フォーマットに従ってドキュメントを生成する
3. `docs/components/{name}.md` に書き出す
4. AskUserQuestion でコミットするか確認する
