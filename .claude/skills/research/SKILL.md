$ARGUMENTS について調査してください。

## 調査プロセス

### 1. 調査対象の特定

ユーザーの指示から以下を抽出する:

- 比較対象のシステム・ライブラリ（複数）
- 比較項目・観点
- 出力ファイルパス（指定がなければ `docs/research/` 配下に適切な名前で作成）

### 2. 並行リサーチ

比較対象ごとに Task エージェント（`general-purpose`）を **並行で** 起動する。
各エージェントへの指示には以下を含める:

- 調査対象の名称とバージョン
- ユーザーが指定した比較項目すべて
- 公式ドキュメント・GitHub リポジトリ・ソースコードの確認を求める指示
- 具体的なコード例・トークン名・設定例の取得を求める指示

### 3. 比較レポートの作成

各エージェントの結果を統合し、以下の構成でマークダウンファイルを作成する:

```
# タイトル

> 調査日・対象バージョン

## サマリー（比較表）
全体像を把握できる表形式のサマリー

## 比較項目ごとの詳細
各項目について、システムごとの詳細な説明・コード例

## 設計判断に向けた考察
自プロジェクトへの適用を検討するための分析

## 参考リンク
各システムの公式ドキュメント・リポジトリへのリンク
```

### 4. 保存とコミット

- `docs/research/` ディレクトリに保存する
- ファイル名はケバブケースで内容を表す名前にする（例: `token-structure-comparison.md`）
- AskUserQuestion でコミットするか確認する

## 品質基準

- 各システムの **実際のソースコード** に基づく具体例を含めること
- 抽象的な説明だけでなく、コード例・命名例を必ず含めること
- 自プロジェクトでの採用判断に使える考察を含めること
- 調査日と対象バージョンを明記すること
